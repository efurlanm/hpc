Implicit SAVE Attribute

The SAVE attribute is given implicitly if:

  • an object, which is not a dummy argument or a PARAMETER, appears in an initialising declaration in a procedure: 

        REAL :: s = 3.5 ! set initially to 3.5

    The equals sign implicitly places the object in the static storage class.

  • an object or part of an object is initialised in a DATA statement, 
        REAL, DIMENSION(50) :: Y
    DATA Y(21:25) /1,2,3,4,5/

    Appearance in a DATA statement implicitly places the object in the static storage class.

  • an object is in COMMON. 

        COMMON /saved/ Y

    Appearance in a COMMON statement implicitly places the object in the static storage class.

  • the procedure (as a whole) has the SAVE attribute. (A SAVE statement on a line on its own means all of the declarations in the procedure have the SAVE attribute.) 

        SUBROUTINE polo
     IMPLICIT NONE
     INTEGER mint, neck_jumper
     SAVE
     REAL stick, car
      ...

    In the above example mint, neck_jumper, stick and car all have the SAVE attribute.

In a procedure, objects are conceptually initialised on the first call only; on all subsequent executions they retain the values that they were assigned during the previous invocation of the procedure.
